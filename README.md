# ChatAnalysis.SKILL

[English](README_en-US.md) | 中文

聊天记录深度分析工具。输入原始聊天数据，输出结构化统计报告、人格画像、话题专题、精彩发言展示，全部渲染为可浏览的 HTML 页面。

## 它能做什么

```
原始聊天数据 ──→ 统计解析 ──→ 深度分析报告 ──→ Markdown（核心）
                  │              │                │
                  │              ├─ 统计分析(5篇)   └─→ HTML（推荐渲染）
                  │              ├─ 综合报告(4篇)        ├─ 5 种预设主题
                  │              ├─ 话题专题(N篇)        ├─ 浅色/暗色主题
                  │              ├─ 人格画像(每人)        ├─ TOP N 排行榜
                  │              ├─ 精彩/神人发言         ├─ 面包屑导航
                  │              └─ 群聊历险记(可选)      └─ 响应式布局
                  └─ 消息统计、时间分布、词频分析、
                     互动模式、回复速度、活动断层
```

## 支持的聊天场景

| 场景 | 说明 | 特色 |
|------|------|------|
| **私聊** | 两人一对一 | 关系动态分析、双方对比画像 |
| **群聊** | 多人群组 | 社交网络、活跃排行、Bot 互动、梗文化 |

## 支持的数据格式

**不做格式限制。** 只要数据包含时间、人物、对话三要素，即可分析。脚本根据实际数据格式在运行时生成。

常见格式示例：

| 格式 | 说明 |
|------|------|
| JSONL 批量文件 | 结构化 JSON Lines（如 QQ Chat Exporter 导出），含 `timestamp` + `sender` 字段 |
| 纯文本 | 各平台聊天记录文本，含日期头和发送人前缀（如 `## 2026-01-01 09:30` / `用户A：内容`） |

## 设计理念

### 广度优先，深度并行

先广泛扫描全貌（统计、时间、内容、互动、媒体、话题、用户画像），再用子代理并行处理各维度的深度报告。

### 善用子代理

任务量大、需要并行时，开子代理是最高效的方式。主对话负责决策、确认、交互、审查；子代理负责执行、生成、转换、批量操作。小任务直接在主对话中完成即可。

### 就事论事，理性优先

每个结论必须有聊天记录原文支撑。客观描述可观察行为，不预设立场，不代入道德判断。使用限定性语言（"从聊天中可观察到"、"在此场景中表现出"）。

### 相对路径原则（T0）

所有输出中的路径必须使用相对路径。禁止绝对路径、`file://` 协议、硬编码域名。页面内 `#id` 锚点跳转使用裸 `#id` 格式。

### 局部视角，不以偏概全

聊天记录只是用户生活的片面切片。对任何用户的分析都只是局部的、缺乏现实生活数据的。主观性输出（锐评、精彩发言等）需避免以偏概全的论断。

## 如何使用

> **TOKEN 消耗警告：** 本 SKILL 会消耗大量 TOKEN（动辄百万），请确保能承担再使用。

将本 SKILL 注册到你的 Agent 平台后，触发聊天分析任务即可。以 Claude Code 为例：

```
/chat-analysis
```

然后按提示逐步配置（Phase 0.0 数据路径确认 + Phase 0.3 配置项，群聊额外 4 项）：

**前置步骤：聊天数据路径** — 文件或目录（Phase 0.0 目录初始化时确认）

Phase 0.3 配置项：

1. **聊天平台** — QQ、微信、Telegram、Discord、WhatsApp，或自动检测
2. **聊天模式** — 私聊或群聊（可自动检测）
3. **焦点用户** — 需深度分析的用户（通常是自己的账号）
4. **输出目录** — 结果写入路径
5. **输出模式** — `md`（仅核心）、`html`（核心+渲染，默认推荐）、`all`（核心+渲染+保留 md）
6. **HTML 风格主题** — 5 种预设视觉风格（仅 HTML/ALL 模式时询问）
7. **HTML 页面布局** — 4 种排版结构（仅 HTML/ALL 模式时询问）
8. **完整聊天记录长页** — 可选，默认不生成。敏感内容用 `<details>` 隐藏，媒体用链接插入，消息按 5 分钟分组合并显示
9. **页脚技术支持链接** — 可选，是否在 HTML 页脚显示项目 GitHub 链接
10. **叙述人称** — 第三人称（默认）或第一人称
11. **敏感内容判断范围** — 仅媒体文件（默认）或所有内容
12. **敏感内容大赏** — 可选，需主动要求开启
13. **最大并行子代理数** — 私聊默认 3，群聊默认 5，最大 9
14. **头像嵌入** — 可选，如原始数据含头像

### 群聊额外选项

群聊模式下会额外询问：

15. **TOP N 排名** — 为排名前 N 的用户单独生成人格画像（默认 10）
16. **more/ 功能性单独页** — 多选，逐项确认：精彩发言（20-30 条）、神人发言（10-20 条），默认全不选
17. **群聊历险记** — 以不同用户的第一人称视角撰写历险日记（默认不生成）
18. **真实身份数据** — 群聊中是否允许出现真实身份（真名、手机号等），默认脱敏

### 你会得到什么

**私聊：**
- 首页大标题（双方昵称，如"张三 & 李四 的聊天分析"）
- 双方统计对比
- 关系动态分析
- 话题专题报告
- 双人性格画像

**群聊：**
- 首页大标题（群聊名称）+ TOP N 活跃排行（可点击跳转画像）
- 每人性格画像
- 话题专题报告
- 精彩发言 / 神人发言展示页（可选）
- 活动断层检测（Tier 分层）
- 群聊历险记（可选）

## 可选玩法模块

以下模块默认关闭，需要你在任务启动时主动开启：

### 精彩发言 / 神人发言

群聊模式自动提议生成。精彩发言 20-30 条，神人发言 10-20 条，带深度解析和梗文化溯源。

### 群聊历险记

以不同用户的第一人称视角撰写群聊历险日记。每篇 150-250 行，基于真实聊天事件，创意但不虚构。严格标注 AI 生成声明。目录下包含 `index.html` 入口页（用户列表/导航）和各用户的独立页面。

### 完整聊天记录长页

生成全部消息的单页 HTML（`html/.whole/chat.html`），可按时间线浏览完整对话。消息按 5 分钟分组合并显示，敏感内容用 `<details>` 独立收起，媒体文件用链接插入（不内嵌），含页脚链接（如用户允许）。该目录完全自包含，可独立部署。

### TOP N 排行榜 + 活动断层检测

群聊模式下生成活跃排行，前三名高亮，点击可跳转人格画像。自动检测活跃度断层（Tier 分层），区分核心活跃、中度参与和潜水用户。

### HTML 风格主题

5 种预设视觉风格，任务启动时选择：

| 主题 | 风格 | 强调色 |
|------|------|--------|
| Apple（默认） | 苹果风格，简洁明亮 | `#007aff` 蓝色 |
| Nord | 北欧冷色调，柔和低饱和 | `#88c0d0` 霜蓝 |
| Dracula | 暗紫风格，深色为主 | `#bd93f9` 紫色 |
| Gruvbox | 复古暖色调，怀旧质感 | `#d79921` 金色 |
| Solarized | 平衡低对比度，护眼配色 | `#268bd2` 青色 |

### HTML 页面布局

4 种排版结构，任务启动时选择：

| 布局 | 最大宽度 | 导航方式 | 说明 |
|------|----------|----------|------|
| Standard（默认） | 880px | 面包屑 | 单栏居中，适合大多数场景 |
| Wide | 1200px | 右侧 TOC | 宽屏布局，适合长文报告 |
| Magazine | 880px | 面包屑 | 首页双栏卡片网格，适合多主题展示 |
| Compact | 880px | 面包屑 | 紧凑密集，适合大量内容浏览 |

## 输出页面特性

- **纯前端** — 技术栈严格限制为 HTML + CSS + JS，无外部依赖
- **独立部署** — 每个 `.xxx/` 目录完全自包含（.assets + resources + 页面），可单独部署到任意静态服务器
- **浅色/暗色** — 一键切换，跨页面持久化
- **相对路径** — 所有链接使用相对路径，可随意迁移部署
- **响应式** — 移动端适配，768px 断点

## 注意事项

### TOKEN 消耗

本 SKILL 会消耗大量 TOKEN（动辄百万），请确保能承担再使用。分析耗时取决于消息量：1K 条以下约 2-5 分钟，1K-5K 条约 5-10 分钟，5K-25K 条约 10-20 分钟，25K 条以上约 20-30 分钟。

### 敏感内容处理

分析输出存在多种形式目录，对应不同的敏感内容处理方式：

- **`.full/`** — 完整版（默认）。严重违法/色情内容自动用 `<details>` 收起隐藏（HTML）或注释标记包裹（Markdown）；一般敏感内容（争议性言论等）保留在正文中，添加视觉警告（md 引用块 / HTML 红边框）
- **`.pure/`** — 纯净版。自动移除严重敏感块，由 `.full/` 派生
- **`.sensitive/`** — 敏感标记/敏感内容大赏（可选）。默认仅放置敏感标记，用户开启"敏感内容大赏"后升级为完整内容摘录
- **`.parts/`** — 分块输出（可选）。完整内容打碎为分块，属"敏感内容大赏"模块
- **`.whole/`** — 完整聊天记录长页（可选）。全部消息单页 HTML，可独立部署

默认仅对媒体文件做敏感判断，可选扩展到所有内容。媒体文件的敏感性需结合聊天上下文判断。

### 隐私

- 群聊中的人物真实身份（真名、手机号等）默认脱敏处理
- 私聊中双方身份允许出现（双方已知）
- 任务启动时会询问你是否允许出现真实身份数据

### 数据要求

- 一次任务的所有数据必须来自**同一个私聊/群聊**，禁止跨聊天混合
- 任务中途可追加新的聊天记录数据，但需确认属于同一聊天（增量数据支持）
- 如需分析多个聊天，应分别启动独立任务

### 叙述视角

- 默认第三人称叙事（不出现"我"）
- 可选择第一人称（需指定"我"对应哪个账号）

### 关键规则

- 不要在任务过程中执行 git 操作（commit/push/merge 等），除非用户明确要求
- 所有输出路径使用相对路径，禁止绝对路径和 `file://` 协议

## 常见问题

**Q: Windows 上 Python 命令用什么？**
用 `python` 而不是 `python3`。

**Q: 分析大概需要多久？**
取决于消息量和子代理数：1K 条以下约 2-5 分钟，1K-5K 条约 5-10 分钟，5K-25K 条约 10-20 分钟，25K 条以上约 20-30 分钟。

**Q: Bot 账号消息量太大怎么办？**
可考虑从"活跃用户"排行中排除 Bot，单独分析。

**Q: 输出可以部署到网页吗？**
可以。所有输出为纯前端 HTML，使用相对路径，直接上传到任意静态托管即可。
