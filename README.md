# ChatAnalysis.SKILL

[English](README_en-US.md) | 中文

聊天记录深度分析工具。输入原始聊天数据，输出结构化统计报告、人格画像、话题专题、精彩发言展示，全部渲染为可浏览的 HTML 页面。

## 它能做什么

```
原始聊天数据 ──→ 统计解析 ──→ 深度分析报告 ──→ Markdown（核心）
                  │              │                │
                  │              ├─ 统计分析(5篇)   └─→ HTML（推荐渲染）
                  │              ├─ 综合报告(4篇)        ├─ 5 种预设主题
                  │              ├─ 话题专题(N篇)        ├─ 浅色/暗色主题
                  │              ├─ 人格画像(每人)        ├─ TOP N 排行榜
                  │              ├─ 精彩/神人发言         ├─ 面包屑导航
                  │              └─ 群聊历险记(可选)      └─ 响应式布局
                  └─ 消息统计、时间分布、词频分析、
                     互动模式、回复速度、活动断层
```

## 支持的聊天场景

| 场景 | 说明 | 特色 |
|------|------|------|
| **私聊** | 两人一对一 | 关系动态分析、双方对比画像 |
| **群聊** | 多人群组 | 社交网络、活跃排行、Bot 互动、梗文化 |

## 支持的数据格式

**不做格式限制。** 只要数据包含时间、人物、对话三要素，即可分析。脚本根据实际数据格式在运行时生成。

常见格式示例：

| 格式 | 说明 |
|------|------|
| JSONL 批量文件 | 结构化 JSON Lines（如 QQ Chat Exporter 导出），含 `timestamp` + `sender` 字段 |
| 纯文本 | 各平台聊天记录文本，含日期头和发送人前缀（如 `## 2026-01-01 09:30` / `用户A：内容`） |

## 安装

### 前置条件

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) CLI 已安装并可正常使用

### 建议环境

- Python 3.8+（统计解析和 HTML 渲染脚本会在运行时生成，建议安装）
- `pip install markdown`（HTML 渲染时需要，运行时会提示安装）

> 本 SKILL 是通用规范，不绑定特定前置环境。Agent 会根据实际可用环境调整脚本实现。

### 安装步骤

```bash
# 克隆仓库
git clone https://github.com/JularDepick/ChatAnalysis.SKILL.git
cd ChatAnalysis.SKILL
```

## 如何使用

在 Claude Code 中输入：

```
/chat-analysis
```

然后按提示逐步配置：

1. 聊天数据路径（文件或目录）
2. 聊天模式（私聊/群聊，可自动检测）
3. 焦点用户（你自己的账号）
4. 输出目录与输出风格（HTML / Markdown / 全部）
5. HTML 视觉风格（5 种预设主题）和页面布局（4 种模板）
6. 完整聊天记录长页（可选，默认不生成）
7. 页脚技术支持链接（可选，是否在 HTML 页脚显示项目链接）
8. 叙述人称（第三人称 / 第一人称）
9. 敏感内容判断范围（仅媒体文件 / 所有内容）
10. 敏感内容大赏（可选，需主动要求开启）
11. 最大并行子代理数（报告生成时的并行上限，默认 3，最大 9）
12. 头像嵌入（可选，如原始数据含头像）

### 群聊额外选项

群聊模式下会额外询问：

- **TOP N** — 为排名前 N 的用户生成人格画像（默认 10，最多 20）
- **精彩发言 / 神人发言** — 是否生成精选展示页
- **群聊历险记** — 是否以不同用户的第一人称视角撰写历险日记
- **最大并行子代理数** — 报告生成时的最大并行子代理数（默认 3，最大 9）

### 你会得到什么

**私聊：**
- 首页大标题（双方昵称，如"张三 & 李四 的聊天分析"）
- 双方统计对比
- 关系动态分析
- 话题专题报告
- 双人性格画像

**群聊：**
- 首页大标题（群聊名称）+ TOP N 活跃排行（可点击跳转画像）
- 每人性格画像
- 话题专题报告（9-12 篇典型）
- 精彩发言 / 神人发言展示页
- 活动断层检测（Tier 分层）
- 群聊历险记（可选）

    ├── .cache/                   ← 临时文件/中间产物（任务完成后清理）
    ├── md/                       ← Markdown 报告（核心输出）
    │   ├── *.md                  ← 默认扁平放置
    │   ├── analyse/              ← 统计分析（用户要求分组时）
    │   ├── report/               ← 综合报告

        │   ├── .assets/          ← CSS/JS
        │   ├── resources/        ← 媒体文件
        │   ├── index.html        ← 首页 + 排行榜

        │   ├── SpeakAs/          ← 群聊历险记
        │   └── more/             ← 单页玩法
        │       ├── brilliant.html

## 可选玩法模块

以下模块默认关闭，需要你在任务启动时主动开启：

### 精彩发言 / 神人发言

群聊模式自动提议生成。精彩发言 20-30 条，神人发言 10-20 条，带深度解析和梗文化溯源。

### 群聊历险记 (SpeakAs/)

以不同用户的第一人称视角撰写群聊历险日记。每篇 150-250 行，基于真实聊天事件，创意但不虚构。严格标注 AI 生成声明。

### 完整聊天记录长页

生成全部消息的单页 HTML，可按时间线浏览完整对话。消息按 5 分钟分组合并显示，敏感内容用 `<details>` 独立收起。媒体文件用链接插入（不内嵌），含页脚链接（如用户允许）。输出到 `html/.completed-whole-page/`，可独立部署。

### TOP N 排行榜 + 活动断层检测

群聊模式下生成活跃排行，前三名高亮，点击可跳转人格画像。自动检测活跃度断层（Tier 分层），区分核心活跃、中度参与和潜水用户。

### HTML 风格主题

5 种预设视觉风格，任务启动时选择：

| 主题 | 风格 | 强调色 |
|------|------|--------|
| Apple（默认） | 苹果风格，简洁明亮 | `#007aff` 蓝色 |
| Nord | 北欧冷色调，柔和低饱和 | `#88c0d0` 霜蓝 |
| Dracula | 暗紫风格，深色为主 | `#bd93f9` 紫色 |
| Gruvbox | 复古暖色调，怀旧质感 | `#d79921` 金色 |
| Solarized | 平衡低对比度，护眼配色 | `#268bd2` 青色 |

### HTML 页面布局

4 种排版结构，任务启动时选择：

| 布局 | 最大宽度 | 导航方式 | 说明 |
|------|----------|----------|------|
| Standard（默认） | 880px | 面包屑 | 单栏居中，适合大多数场景 |
| Wide | 1200px | 右侧 TOC | 宽屏布局，适合长文报告 |
| Magazine | 880px | 面包屑 | 首页双栏卡片网格，适合多主题展示 |
| Compact | 880px | 面包屑 | 紧凑密集，适合大量内容浏览 |

## 输出页面特性

- **纯前端** — 技术栈严格限制为 HTML + CSS + JS，无外部依赖
- **独立部署** — 每个 `.xxx/` 目录完全自包含（.assets + resources + 页面），可单独部署到任意静态服务器
- **浅色/暗色** — 一键切换，跨页面持久化
- **相对路径** — 所有链接使用相对路径，可随意迁移部署
- **响应式** — 移动端适配，768px 断点

## 注意事项

### 敏感内容处理

- 严重违法/色情内容自动用 `<details>` 收起隐藏（HTML），Markdown 中以注释标记包裹
- 一般敏感内容（争议性言论等）保留在正文中，添加视觉警告（md 引用块 / HTML 红边框）
- 默认仅对媒体文件做敏感判断，可选扩展到所有内容
- 纯净版（`.pure/`）自动移除敏感块，完整版（`.completed-hidden/`）保留但隐藏
- 可选"敏感内容大赏"模块，需主动要求开启

### 隐私

- 群聊中的人物真实身份（真名、手机号等）默认脱敏处理
- 私聊中双方身份允许出现（双方已知）
- 任务启动时会询问你是否允许出现真实身份数据

### 数据要求

- 一次任务的所有数据必须来自**同一个私聊/群聊**，禁止跨聊天混合
- 任务中途可追加新的聊天记录数据，但需确认属于同一聊天
- 如需分析多个聊天，应分别启动独立任务

### 叙述视角

- 默认第三人称叙事（不出现"我"）
- 可选择第一人称（需指定"我"对应哪个账号）

## 常见问题

**Q: Windows 上 Python 命令用什么？**
用 `python` 而不是 `python3`。

**Q: 分析大概需要多久？**
取决于消息量和子代理数：1K 条以下约 2-5 分钟，1K-5K 条约 5-10 分钟，5K-25K 条约 10-20 分钟，25K 条以上约 20-30 分钟。

**Q: Bot 账号消息量太大怎么办？**
可考虑从"活跃用户"排行中排除 Bot，单独分析。

**Q: 输出可以部署到网页吗？**
可以。所有输出为纯前端 HTML，使用相对路径，直接上传到任意静态托管即可。
