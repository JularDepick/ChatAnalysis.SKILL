# Script Writing Guide for Chat Analysis

This document contains patterns, conventions, and best practices for writing analysis scripts at runtime. Scripts are NOT pre-built — they are generated fresh for each task based on the actual data format.

## Core Principle

> Every chat export has subtle format differences. The script must adapt to the data, not the other way around.
> Read the actual data first, then write the script.

---

## 1. Parser Script Patterns

### 1.1 Text Format Parser

#### Detection Phase

Before writing the parser, read the first 100-200 lines of the chat file to identify:

1. **Date header pattern** — How are dates/times marked?
   - `## YYYY-MM-DD HH:MM` (QQ export)
   - `YYYY-MM-DD HH:MM:SS` (WeChat export)
   - `[YYYY.MM.DD HH:MM:SS]` (Telegram export)
   - `MM/DD/YYYY, HH:MM -` (WhatsApp export)
   - Custom format — build regex accordingly

2. **Sender pattern** — How are senders identified?
   - `Name：content` (Chinese colon)
   - `Name: content` (English colon)
   - `Name\ncontent` (newline-separated, WeChat style)
   - `Name —` (Discord style)

3. **Media markers** — How are non-text messages marked?
   - `[图片]`, `[Image]`, `[Photo]`
   - `[卡片消息: title]`
   - `[表情]`, `[Sticker]`
   - `[文件]`, `[File]`

4. **System messages** — What patterns indicate system events?
   - `撤回了一条消息` (message recall)
   - `加入了群聊` (join group)
   - `红包` (red envelope)

#### Format Detection via Scoring

当数据来源不明时，用评分机制自动检测格式。读取前 100 行，对每种已知格式的正则匹配计分，取最高分：

```python
FORMAT_PATTERNS = {
    "qq": {
        "date": r"^## (\d{4}-\d{2}-\d{2}) (\d{2}:\d{2})$",
        "sender": r"^([^：]+)：(.*)$",
        "image": "[图片]",
        "card": r"^\[卡片消息:.*\]$",
    },
    "wechat": {
        "date": r"^(\d{4}-\d{2}-\d{2}) (\d{2}:\d{2}:\d{2})$",
        "sender": r"^([^：:]+)[：:](.*)$",
        "image": "[图片]",
        "card": None,
    },
    "telegram": {
        "date": r"^\[(\d{4}\.\d{2}\.\d{2}) (\d{2}:\d{2}:\d{2})\]$",
        "sender": r"^([^:]+): (.*)$",
        "image": "[Photo]",
        "card": None,
    },
    "generic": {
        "date": r"^(\d{4}-\d{2}-\d{2}) (\d{2}:\d{2})",
        "sender": r"^([^：:]+)[：:](.*)$",
        "image": "[图片]",
        "card": None,
    },
}

def detect_format(filepath):
    """Auto-detect chat format from first 100 lines.
    Score: date header match = +3, sender match = +1.
    """
    with open(filepath, "r", encoding="utf-8") as f:
        lines = [f.readline() for _ in range(100)]

    scores = {}
    for fmt, patterns in FORMAT_PATTERNS.items():
        score = 0
        for line in lines:
            line = line.strip()
            if re.match(patterns["date"], line):
                score += 3
            elif re.match(patterns["sender"], line):
                score += 1
        scores[fmt] = score

    return max(scores, key=scores.get)
```

**要点：** 日期行权重高于发送人行（日期格式更具区分性）。新平台适配时只需往 `FORMAT_PATTERNS` 添加新条目。

#### Parser Template

```python
#!/usr/bin/env python3
"""
Chat Log Parser — Written for: <describe the actual data>
Auto-generated by chat-analysis skill.
"""
import re
import json
import os
import sys
from collections import Counter, defaultdict
from datetime import datetime, timedelta

# === ADAPT THESE PATTERNS TO THE ACTUAL DATA ===
DATE_RE = r"^## (\d{4}-\d{2}-\d{2}) (\d{2}:\d{2})$"
SENDER_RE = r"^([^：]+)：(.*)$"
IMAGE_TAG = "[图片]"
CARD_TAG_RE = r"^\[卡片消息:.*\]$"

def parse_chat(filepath, author_name=None):
    messages = []
    current_date = None
    current_time = None

    with open(filepath, "r", encoding="utf-8") as f:
        for line_num, line in enumerate(f, 1):
            line = line.rstrip("\n")

            # Check for date header
            date_match = re.match(DATE_RE, line)
            if date_match:
                current_date = date_match.group(1).replace(".", "-")
                current_time = date_match.group(2)
                continue

            # Check for sender message
            sender_match = re.match(SENDER_RE, line)
            if sender_match and current_date:
                sender = sender_match.group(1).strip()
                content = sender_match.group(2).strip()

                # Normalize "me" label
                if author_name and sender == author_name:
                    sender = "我"

                # Classify message type
                msg_type = classify_message(content)

                # Build timestamp
                ts = build_timestamp(current_date, current_time)

                messages.append({
                    "date": current_date,
                    "time": current_time,
                    "sender": sender,
                    "content": content,
                    "msg_type": msg_type,
                    "timestamp": ts.isoformat() if ts else None,
                    "line": line_num,
                })

    return messages

def classify_message(content):
    if not content:
        return "empty"
    if content.strip() == IMAGE_TAG:
        return "image"
    if CARD_TAG_RE and re.match(CARD_TAG_RE, content):
        return "card"
    if re.match(r"^\[.+\]$", content.strip()):
        return "system"
    return "text"

def build_timestamp(date_str, time_str):
    for fmt in ["%Y-%m-%d %H:%M", "%Y-%m-%d %H:%M:%S"]:
        try:
            return datetime.strptime(f"{date_str} {time_str}", fmt)
        except ValueError:
            continue
    return None
```

### 1.2 JSONL Format Parser

#### Detection Phase

Read 5-10 lines of a `.jsonl` file to identify:

1. **Top-level fields** — What keys exist at the root?
   - `id`, `seq`, `timestamp`, `time`, `sender`, `type`, `content`, `recalled`, `system`

2. **Sender object structure** — How is sender info nested?
   - `sender.uin` — QQ number (stable identifier)
   - `sender.uid` — Internal UID
   - `sender.name` — Display name
   - `sender.groupCard` — Group nickname (may change)

3. **Content object structure** — How is content nested?
   - `content.text` — Plain text
   - `content.html` — HTML formatted
   - `content.elements` — Structured elements array
   - `content.resources` — Media resources

4. **Timestamp format** — How is time represented?
   - `timestamp` field: milliseconds since epoch
   - `time` field: ISO 8601 string

5. **Message types** — What values does `type` field have?
   - `text`, `image`, `forward`, `reply`, `system`

#### Parser Template

```python
#!/usr/bin/env python3
"""
JSONL Chat Parser — Written for: <describe the actual data>
Auto-generated by chat-analysis skill.
"""
import json
import os
import sys
import re
from collections import Counter, defaultdict
from datetime import datetime, timedelta

def load_all_messages(batch_dir):
    """Load all messages from JSONL batch files."""
    messages = []
    batch_files = sorted([
        f for f in os.listdir(batch_dir)
        if f.endswith(".jsonl")
    ])

    for bf in batch_files:
        filepath = os.path.join(batch_dir, bf)
        with open(filepath, "r", encoding="utf-8") as f:
            for line_num, line in enumerate(f, 1):
                line = line.strip()
                if not line:
                    continue
                try:
                    msg = json.loads(line)
                    messages.append(msg)
                except json.JSONDecodeError:
                    print(f"  Warning: Skip invalid JSON in {bf}:{line_num}")

    return messages

def get_sender_name(msg):
    """Get best display name for a sender.
    Resolution order: groupCard > name > nickname
    """
    sender = msg.get("sender", {})
    return (sender.get("groupCard")
            or sender.get("name")
            or sender.get("nickname")
            or "unknown")

def get_sender_uin(msg):
    """Get sender's stable identifier (QQ number)."""
    return msg.get("sender", {}).get("uin", "unknown")

def parse_timestamp(msg):
    """Parse timestamp from message.
    Handles both ms-since-epoch and ISO string formats.
    """
    ts_ms = msg.get("timestamp")
    if ts_ms:
        return datetime.fromtimestamp(ts_ms / 1000)
    time_str = msg.get("time", "")
    if time_str:
        try:
            return datetime.fromisoformat(
                time_str.replace("Z", "+00:00")
            ).replace(tzinfo=None)
        except:
            pass
    return None

def classify_message(msg):
    """Classify message type from JSONL structure."""
    if msg.get("recalled"):
        return "recalled"
    if msg.get("system"):
        return "system"
    msg_type = msg.get("type", "text")
    if msg_type == "text":
        text = msg.get("content", {}).get("text", "")
        if text.startswith("[图片"):
            return "image"
    return msg_type

def extract_text(msg):
    """Extract plain text content."""
    return msg.get("content", {}).get("text", "")
```

## 2. Statistics Computation Patterns

### 2.1 Basic Statistics

```python
def compute_basic_stats(messages, valid_msgs):
    dates = sorted(set(
        parse_timestamp(m).strftime("%Y-%m-%d")
        for m in valid_msgs if parse_timestamp(m)
    ))
    senders = Counter(get_sender_uin(m) for m in valid_msgs)
    msg_types = Counter(classify_message(m) for m in valid_msgs)

    return {
        "total_messages": len(messages),
        "valid_messages": len(valid_msgs),
        "date_range": {"start": dates[0], "end": dates[-1]} if dates else None,
        "active_days": len(dates),
        "total_days": (
            (datetime.strptime(dates[-1], "%Y-%m-%d")
             - datetime.strptime(dates[0], "%Y-%m-%d")).days + 1
            if dates else 0
        ),
        "senders": dict(senders.most_common()),
        "sender_names": {uin: get_sender_name(m) for ...},
        "message_types": dict(msg_types),
        "dates": dates,
    }
```

def compute_time_stats(valid_msgs):
    hourly = defaultdict(int)
    weekday = defaultdict(int)
    monthly = defaultdict(int)
    daily = defaultdict(int)

    for m in valid_msgs:
        ts = parse_timestamp(m)
        if ts:
            hourly[ts.hour] += 1
            weekday[ts.strftime("%A")] += 1
            monthly[ts.strftime("%Y-%m")] += 1

        "hourly": dict(sorted(hourly.items())),
        "weekday": dict(weekday),
        "monthly": dict(sorted(monthly.items())),
        "daily": dict(sorted(daily.items())),
    }

### 2.3 Chinese Word Extraction (No jieba)

```python
CHINESE_STOP_WORDS = set(
    "的了是在我你他她它们这那就也都有不人一个上来到大为和与对中说会要"
    "能把可以还会没有很什么被让把给到从用得着过下去出来已经"
    "吧吗呢啊哦嗯呀哈哇唉嘛喂"
    "一二三四五六七八九十百千万亿"
    "那这个么什怎哪几多"
    "就是不是这个那个还是而且但是因为所以如果虽然"
)

def extract_chinese_words(text, min_len=2, max_len=4):
    """Simple Chinese word extraction by character n-grams.
    No external dependencies (jieba-free).
    """
    chinese = re.findall(r"[一-鿿]+", text)
    words = []
    for segment in chinese:
        for n in range(min_len, max_len + 1):
            for i in range(len(segment) - n + 1):
                word = segment[i:i + n]
                if word not in CHINESE_STOP_WORDS:
                    words.append(word)
    return words
```

### 2.4 English Word Extraction

```python
def extract_english_words(text):
    """Extract English words (2+ chars), lowercased."""
    return [w.lower() for w in re.findall(r"[a-zA-Z]{2,}", text) if len(w) >= 2]
```

与中文词提取配合使用，合并为完整的词频统计。按用户分组统计可生成个人兴趣画像。

### 2.7 Conversation Round Detection

```python
def detect_rounds(sorted_msgs, gap_seconds=1800):
    """Split messages into conversation rounds.
    A new round starts when there's a gap > gap_seconds (default 30 min).
    """
    rounds = []
    current_round = []
    prev_ts = None

    for m in sorted_msgs:
        ts = parse_timestamp(m)
        if prev_ts and (ts - prev_ts).total_seconds() > gap_seconds:
            if current_round:
                rounds.append(current_round)
            current_round = []
        current_round.append(m)
        prev_ts = ts
    if current_round:
        rounds.append(current_round)

    return rounds
```

### 2.8 Reply Speed Computation

```python
def compute_reply_speed(sorted_msgs):
    """Compute reply speed between consecutive messages from different senders.
    Only considers replies within 1 hour.
    """
    reply_times = defaultdict(list)

    for i in range(1, len(sorted_msgs)):
        curr = sorted_msgs[i]
        prev = sorted_msgs[i - 1]
        curr_uin = get_sender_uin(curr)
        prev_uin = get_sender_uin(prev)

        if curr_uin != prev_uin:
            curr_ts = parse_timestamp(curr)
            prev_ts = parse_timestamp(prev)
            if curr_ts and prev_ts:
                delta = (curr_ts - prev_ts).total_seconds()
                if 0 < delta < 3600:
                    reply_times[curr_uin].append(delta)

    return {
        uin: {
            "median": sorted(times)[len(times) // 2],
            "mean": round(sum(times) / len(times), 1),
            "count": len(times),
        }
        for uin, times in reply_times.items()
    }
```

### 2.9 Activity Cliff Detection (Group Chat)

```python
def detect_activity_cliffs(senders_sorted):
    """Find activity cliffs in group chat user rankings.

    A cliff is detected when:
    1. Drop % > 15% AND absolute drop > 80 messages
    2. OR drop is 2.5x larger than average of next 3 drops

    Returns list of (rank, name, count, drop_pct) tuples.
    """
    cliffs = []
    for i in range(len(senders_sorted) - 1):
        curr_name, curr_count = senders_sorted[i]
        next_name, next_count = senders_sorted[i + 1]
        drop = curr_count - next_count
        drop_pct = drop / curr_count * 100

        # Check absolute + percentage threshold
        if drop_pct > 15 and drop > 80:
            cliffs.append((i + 1, curr_name, curr_count, drop_pct))
            continue

        # Check relative threshold (2.5x larger than avg of next 3 drops)
        if i + 3 < len(senders_sorted):
            next_drops = []
            for j in range(i + 1, min(i + 4, len(senders_sorted) - 1)):
                d = senders_sorted[j][1] - senders_sorted[j + 1][1]
                next_drops.append(d)
            if next_drops and drop > 2.5 * (sum(next_drops) / len(next_drops)):
                cliffs.append((i + 1, curr_name, curr_count, drop_pct))

    return cliffs
```

    function apply(t) {

            document.documentElement.setAttribute('data-theme', 'dark');

    });

### 3.7 Media Embedding

报告中引用媒体文件时，**必须使用 HTML 标签**（`<img>`、`<audio>`、`<video>`），不能用纯文本占位符。关键是固定一个维度（width 或 height）到固定 px 值，不超过屏幕较短一边的 50%。

**所有媒体引用必须使用相对路径**，不使用绝对路径或 `file://` 协议，确保输出目录可迁移部署。

**图片：**
```html
<!-- 固定宽度（推荐，大多数场景） -->
<img src="resources/images/photo.jpg" alt="描述" style="max-width: 360px; height: auto; border-radius: 8px;">

<!-- 固定高度（竖图/长图） -->
<img src="resources/images/tall.jpg" alt="描述" style="max-height: 360px; width: auto; border-radius: 8px;">
```

**音频：**
```html
<audio controls style="width: 360px;">
    <source src="resources/audio/voice.mp3" type="audio/mpeg">
</audio>
```

**视频：**
```html
<video controls style="max-width: 360px; border-radius: 8px;">
    <source src="resources/video/clip.mp4" type="video/mp4">
</video>
```

**尺寸约束规则：**
- 取 `min(360px, 50vw)` 作为基准值（360px 在大多数屏幕上 < 50vw）
- 横图固定 width，竖图固定 height，另一维度 `auto` 保持比例
- 不使用百分比宽度（避免在不同布局下溢出）
- 所有媒体标签加 `border-radius: 8px` 保持圆角风格

**Markdown 中的写法：** 在 md 文件中直接写 HTML 标签，不要用 `![alt](url)` 语法（无法控制尺寸）：
```markdown
<img src="resources/images/screenshot.png" alt="截图" style="max-width: 360px; height: auto; border-radius: 8px;">
```

**善用原始媒体：** 报告引用聊天原文时，如果原始消息包含图片/音频/视频，应主动将对应媒体嵌入报告。媒体是聊天的重要组成部分，图文并茂的报告比纯文字更有质感。

**选择性复制：** 只有实际被报告引用的媒体文件才复制到输出 `resources/` 目录。不要全量复制原始数据中的媒体。

def convert_file(md_path, html_path, category, filename, css, cat_names, footer=""):

### 3.11 Index Page Generation

两类索引页：分类索引（列出该分类下所有文件）和主页索引（列出所有分类及其文件）：

```python
def generate_index(category, files_info, css, cat_names, footer=""):
    """分类索引页：列出该分类下所有文章链接。"""
    cat_name = cat_names.get(category, category)
    items = "".join(
        f'<li><a href="{fname}">{title}</a></li>\n'
        for fname, title in sorted(files_info)
    )
    nav = f'<div class="nav"><a href="../index.html">首页</a><span class="sep">›</span>{cat_name}</div>'
    body = f"<h1>{cat_name}</h1>\n<ul>\n{items}\n</ul>"
    return make_page(cat_name, nav, body, css, footer)

def generate_main_index(all_files, css, cat_names, footer=""):
    """主页索引页：列出所有分类及文章。"""
    items = ""
    for category in sorted(all_files.keys()):
        cat_name = cat_names.get(category, category)
        files = all_files[category]
        items += f'<h2><a href="{category}/index.html">{cat_name}</a> ({len(files)}篇)</h2>\n<ul>\n'
        for fname, title in sorted(files):
            items += f'<li><a href="{category}/{fname}">{title}</a></li>\n'
        items += '</ul>\n'
    body = f"<h1>聊天记录深度分析</h1>\n<hr>\n{items}"
    return make_page("聊天记录深度分析", "", body, css, footer)
```

### 3.14 Time Merging for Chat Display

前端展示聊天记录时，合并连续消息的时间显示：5 分钟内无中断的消息归入同一个起始时间戳下。原始数据保留每条消息的完整时间，仅在渲染层合并。

```python
def merge_time_display(messages, gap_seconds=300):
    """将连续消息按时间窗口分组，返回 [(datetime_str, [msgs]), ...]。
    gap_seconds: 超过此秒数则开始新分组（默认 5 分钟）。
    """
    if not messages:
        return []
    groups = []
    current_group = [messages[0]]
    for i in range(1, len(messages)):
        prev_ts = parse_timestamp(messages[i - 1])
        curr_ts = parse_timestamp(messages[i])
        if prev_ts and curr_ts and (curr_ts - prev_ts).total_seconds() > gap_seconds:
            groups.append(current_group)
            current_group = []
        current_group.append(messages[i])
    if current_group:
        groups.append(current_group)

    # 取每组第一条消息的时间作为显示时间
    return [
        (parse_timestamp(g[0]).strftime("%Y-%m-%d %H:%M"), g)
        for g in groups
    ]
```

渲染时，时间戳作为分组标题显示一次，组内消息不再重复显示时间：

```html
<div class="time-group">
    <div class="time-header">2026-03-17 22:30</div>
    <div class="msg">用户A：消息1</div>
    <div class="msg">用户B：消息2</div>
    <div class="msg">用户A：消息3</div>
</div>
<div class="time-group">
    <div class="time-header">2026-03-17 23:15</div>
    <!-- 新分组 -->
</div>
```

**要点：**
- 原始数据不修改，仅渲染层合并
- 分组阈值可调（默认 5 分钟）
- Markdown 报告中同样适用此分组逻辑

### 3.15 Whole-Page Chat Record

生成完整聊天记录的单页 HTML（`html/.whole-page/chat.html`），用于浏览全部原始消息。

**结构：**
```python
def generate_whole_page(messages, css, footer=""):
    """生成完整聊天记录长页 HTML。
    - 消息按时间合并显示（5分钟分组）
    - 敏感内容用 <details> 隐藏
    - 媒体文件用 <a> 链接（不内嵌）
    """
    groups = merge_time_display(messages, gap_seconds=300)

    body_parts = []
    for time_str, msgs in groups:
        body_parts.append(f'<div class="time-group">')
        body_parts.append(f'  <div class="time-header">{time_str}</div>')
        for m in msgs:
            sender = m["sender"]
            content = m["content"]
            msg_type = m.get("msg_type", "text")

            # 敏感内容判断（根据实际标记逻辑）
            if is_sensitive(m):
                body_parts.append(
                    f'  <details><summary>⚠️ 敏感内容（点击展开）</summary>'
                    f'<div class="danger-block">{format_msg(sender, content, msg_type)}</div>'
                    f'</details>'
                )
            else:
                body_parts.append(f'  {format_msg(sender, content, msg_type)}')
        body_parts.append('</div>')

    body = "\n".join(body_parts)
    return make_page("完整聊天记录", "", body, css, footer)


def format_msg(sender, content, msg_type):
    """格式化单条消息。媒体用 <a> 链接。"""
    if msg_type == "image":
        return f'<div class="msg"><span class="sender">{sender}：</span><a href="resources/{content}" target="_blank">[图片: {content}]</a></div>'
    elif msg_type == "card":
        return f'<div class="msg"><span class="sender">{sender}：</span><span class="card">[卡片] {content}</span></div>'
    elif msg_type == "system":
        return f'<div class="msg system">[系统] {content}</div>'
    else:
        return f'<div class="msg"><span class="sender">{sender}：</span>{content}</div>'
```

**要点：**
- 媒体用 `<a>` 链接而非 `<img>` 内嵌，避免页面过大
- 敏感内容用 `<details>` 折叠，与 `.completed-hidden/` 同逻辑
- 使用时间合并显示（3.14 节），减少时间戳噪音

## 4. Platform-Specific Adaptations

### 4.1 QQ

- JSONL format from QCE (QQ Chat Exporter)
- Sender identified by `uin` (QQ number, stable) and `uid` (internal)
- Group nickname in `sender.groupCard`
- Bot accounts common in groups
- Forward messages (`type: "forward"`)
- Reply messages (`type: "reply"`)

### 4.2 WeChat

- Text format, often with `Name\ncontent` (newline separated)
- Timestamp format: `YYYY-MM-DD HH:MM:SS`
- Image tag: `[图片]`
- May have `[动画表情]` for animated stickers

### 4.3 Telegram

- Text format with `[YYYY.MM.DD HH:MM:SS]` timestamps
- Sender: `Name: content`
- `[Photo]`, `[Video]`, `[Document]` media tags
- Reply chains via message ID references

### 4.4 Discord

- Text format with `[YYYY-MM-DD HH:MM]` timestamps
- Sender: `Name —` or `Name:`
- Rich embeds, reactions, threads

### 4.5 WhatsApp

- Text format with `MM/DD/YYYY, HH:MM -` timestamps
- Sender: `Name: content`
- `<Media omitted>` for media messages
- System messages for join/leave/admin changes

## 5. Error Handling Conventions

### 5.1 Windows Compatibility

```python
import sys
# Fix Chinese character encoding on Windows
if sys.platform == "win32":
    sys.stdout.reconfigure(encoding='utf-8')
```

- Use `python` instead of `python3` on Windows
- Use forward slashes in paths when possible
- Handle encoding explicitly: always use `encoding='utf-8'` in file operations

### 5.2 Data Validation

After parsing, validate:
- Total message count > 0
- Date range is non-empty
- At least 1 sender found
- No unexpected zero-message periods

### 5.3 Graceful Degradation

- If timestamp parsing fails, use `None` and skip time-based analysis
- If sender name is missing, use UIN or "unknown"
- If message type is unrecognized, classify as "text"
- Always produce partial results rather than crashing

## 7. Performance Tips

- Load all messages into memory for cross-batch analysis (25K messages ≈ 50MB)
- Use `Counter` and `defaultdict` for efficient aggregation
- Sort messages by timestamp once, then reuse the sorted list
- For Chinese word extraction, limit n-gram range to 2-4 characters
- For HTML rendering, process files sequentially (not in memory all at once)
- Use `os.makedirs(..., exist_ok=True)` to avoid directory creation errors
