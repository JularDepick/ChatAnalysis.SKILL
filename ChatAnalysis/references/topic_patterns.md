## 基于关键词的话题分类

每个话题类别关联一组关键词，用于自动化检测。
通过阅读代表性片段，将关键词适配到实际聊天内容。

### 模板

```python
TOPIC_KEYWORDS = {
    "topic_name": {
        "keywords": ["word1", "word2", ...],
        "description": "该话题涵盖内容的简要描述",
        "subtopics": {
            "subtopic_1": ["specific_keyword1", "specific_keyword2"],
            "subtopic_2": ["specific_keyword3", "specific_keyword4"],
        }
    }
}
```

## 话题发现流程

### 步骤 1：阅读片段

从聊天文件中读取 8-10 个片段，均匀分布在日期范围内：
- 开头（第 1-3 天）
- 25% 处
- 50% 处
- 75% 处
- 末尾（最后 3 天）
- 3-5 个随机位置

对每个片段，记录：
1. 出现了哪些话题
2. 使用了哪些关键词
3. 谁发起了每个话题

### 步骤 2：构建关键词列表

根据发现的话题，创建全面的关键词列表。
包含聊天中发现的变体和网络用语。

### 步骤 3：全量扫描

对所有消息执行关键词匹配，获取话题频率。

### 步骤 4：深度分析

对每个频率显著（占消息数 >5%）的话题：
1. 收集所有相关消息
2. 分组为子话题
3. 分析时间模式
4. 记录情感维度
5. 撰写附证据的报告
