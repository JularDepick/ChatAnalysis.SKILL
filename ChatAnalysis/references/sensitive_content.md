# 敏感内容处理规范

> 本文档集中定义敏感内容的分类、默认处置流程和可选模块。各 Phase 中涉及敏感内容的具体操作均引用本节。
> 与 `SKILL.md`（流程定义）、`frontend_spec.md`（HTML 组件样式）配合使用。

---

## 1. 定义

| 类型 | 是否敏感 | 说明 |
|------|----------|------|
| **严重违反法律法规** | 是 | 媒体文件涉及未成年人、暴力犯罪、毒品、恐怖主义等 |
| **漏点色情** | 是 | 媒体文件暴露关键部位的色情内容 |
| 价值观不同 | 否 | 政治观点、宗教信仰、生活方式差异 |
| 软色情 | 否 | 媒体文件擦边、暗示性内容，但未暴露关键部位 |
| 无露出色情 | 否 | 色情内容但未暴露关键部位 |
| 争议性言论 | 否 | 不当言论、冒犯性表达，按一般敏感内容处理（md 引用块 / HTML 红边框警告） |

**默认仅针对媒体文件。** 文字内容一般不做敏感处理（争议性言论仅加红边框警告，不隐藏不脱敏）。默认敏感判断范围：图片、音频、视频等媒体文件。

**媒体文件的敏感性判断：** 媒体文件是否属于敏感内容，不能仅看文件本身，必须结合其在聊天记录中的上下文判断。同一张图片在不同语境下敏感性可能不同。判断时应参考：文件名、前后消息内容、发送者意图、讨论话题。

**敏感内容判断范围** — 在 Phase 0.3 第 11 项询问用户，选项为"仅媒体文件（默认）"或"所有内容"。

## 2. 默认行为（无需用户提示）

发现严重违法/色情内容时，自动执行以下流程：

### Markdown 输出（核心）

敏感内容在 md 文件中以注释标记包裹，不直接隐藏：
```markdown
<!-- sensitive-start: 严重违法/色情 -->
敏感内容原文
<!-- sensitive-end -->
```
每条敏感内容独立标记，标记对之间一一对应。

### HTML 输出（从 md 渲染时）

渲染器将 `<!-- sensitive-start/end -->` 标记转换为 `<details>` 块：
```html
<details>
  <summary>⚠️ 敏感内容（严重违法/色情）— 点击展开</summary>
  <div class="danger-block">
    <!-- 敏感内容原文 -->
  </div>
</details>
```
**每条敏感内容独立收起/展开，互不影响。** 不得使用联动逻辑（展开一条即展开全部）。每个 `<details>` 块独立运作。

## 3. 形式目录派生

"形式目录"指 `html/` 下的 `.xxx/` 子目录，每个是一种独立的输出展示形式。`.full/` 为基准，其余由此派生。

| 形式目录 | 说明 | 派生方式 |
|----------|------|----------|
| `.full/` | 完整版（默认），敏感内容用 `<details>` 隐藏 | 基准，直接从 md 渲染 |
| `.sensitive/` | 敏感内容大赏模块 | 默认仅放置敏感标记（来源、位置、类型）；用户开启后升级为完整内容摘录 |
| `.pure/` | 纯净版 | 从 `.full/` 移除整个 `<details>` 敏感块 |
| `.parts/` | 分块输出 | 完整内容打碎为分块（纯净分块 + 敏感分块），属"敏感内容大赏"模块 |
| `.whole/` | 完整聊天记录长页 | 可选，默认不生成，用户在 Phase 0.3 第 8 项选择后生成 |

**集合关系：**
- `.full = .pure ∪ 隐藏的敏感块`
- `.pure = .full - 敏感块`

**任务结束后报告** — 向用户说明发现了哪些敏感内容、Agent 做了什么处理、如何查看各版本。

---

## 4. 可选模块：敏感内容大赏（默认关闭，不主动引导）

仅当用户**主动明确要求**时才开启：

1. 填充 `html/.sensitive/` — 从仅标记升级为完整内容摘录
2. 生成 `html/.parts/` — 完整内容打碎为分块（纯净分块 + 敏感分块）
3. 用户可通过分块自由组合：纯净块 → 纯净版，全部块 → 完整版
4. md/ 中的 `<!-- sensitive-start/end -->` 标记保持不变，作为敏感内容的原始来源

**不主动引导用户开启此模块。** 仅在用户自行提出时响应。

---

## 5. 一般敏感内容（非严重）

一般敏感内容（价值观不同、软色情、无露出色情、争议性言论等）— 保留在正文中，添加视觉警告：

1. **禁止修改或审查原文** — 不得删改聊天记录原文的文字内容（隐私脱敏另见隐私保护规则，脱敏操作不受此限制）。用 `<!-- sensitive-start/end -->` 标记包裹不算"修改"，原文内容保持不变
2. **Markdown 中**：在相关内容前添加 `> ⚠️ 以下内容可能包含争议性言论` 引用块警告
3. **HTML 渲染时**：转为红边框警告 div（样式见 `frontend_spec.md` 4.4 节）
4. **应转述而非逐字引用** — 分析评论中转述敏感部分，避免逐字复制
5. **聚焦沟通行为分析** — 分析沟通模式而非认同内容
6. **以学术框架呈现** — 如"从沟通行为分析角度..."

---

## 6. 任务结束报告

任务完成后，Agent 必须向用户报告敏感内容处理情况：

- 发现了哪些敏感内容（类型、数量、大致位置）
- Agent 做了什么处理（标记、隐藏、移除）
- 如何查看各版本（`.full/` 查看完整版，`.pure/` 查看纯净版）
- 如何开启"敏感内容大赏"（如未开启）
